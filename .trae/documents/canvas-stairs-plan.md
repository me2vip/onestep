# Canvas 台阶狠话与真心 - 项目计划

## 1. 需求分析与项目概述

### 1.1 项目目标
创建一个具有现代UI的交互式Canvas网页项目，主题为"狠话与真心"。用户可以在控制台上切换不同的文字类型，配合动态特效和背景音乐，在Canvas画布上呈现"一级台阶"式的视觉效果。

### 1.2 核心需求拆解
| 需求 | 描述 | 技术实现 |
|------|------|----------|
| 收集狠话与真心 | 收录网络上的经典"狠话"（尖锐、毒舌）和"真心"（温暖、真实）语录 | 本地JSON数据，预置20+条精选语录 |
| Canvas绘制台阶效果 | 使用Canvas 2D API绘制台阶（阶梯）图形作为文字承托 | `requestAnimationFrame`循环 + Canvas路径绘制 |
| 控制面板 | 调节文字类型、特效开关、音乐音量等 | HTML表单控件 + 事件绑定 |
| 背景音乐 | 循环播放背景音，可开关、可调节音量 | `<audio>`元素或Web Audio API |
| 动态特效 | 粒子、渐入渐出、发光、波纹等动效 | Canvas粒子系统 + CSS动画 |
| 现代UI | 磨砂玻璃、渐变、深色主题、响应式布局 | CSS3（backdrop-filter, gradients, flex/grid） |

### 1.3 技术栈
- **纯前端实现**，无需构建工具
- HTML5 + CSS3 + ES6+ 原生JavaScript
- Canvas 2D API（核心渲染）
- CSS3 动画与过渡
- `<audio>` 标签播放音乐

## 2. 项目结构设计

```
really/
├── index.html              # 主页面：Canvas画布 + 控制面板
├── css/
│   └── style.css           # 现代化UI样式（磨砂玻璃、渐变、响应式）
├── js/
│   ├── data.js             # 狠话/真心语录数据
│   ├── canvas.js           # Canvas绘制逻辑（台阶 + 文字 + 粒子）
│   ├── audio.js            # 背景音乐控制
│   └── app.js              # 主入口：事件绑定、状态管理
└── assets/
    └── (可选)              # 如提供本地音频文件可放于此
```

### 2.1 文件职责说明
- **index.html** — 页面骨架：`#app` 容器，`<canvas id="stage">` 画布，`<aside id="panel">` 控制面板，`<audio id="bgm">` 音乐元素
- **css/style.css** — 深色主题 + 霓虹/渐变点缀，控制面板使用毛玻璃效果
- **js/data.js** — 导出 `harshWords[]` 和 `trueWords[]` 两个数组，每条包含 `text` 和 `author`
- **js/canvas.js** — `CanvasScene` 类：绘制台阶3D透视效果，文字沿台阶排布，粒子特效
- **js/audio.js** — `AudioController` 类：加载/播放/暂停/音量调节
- **js/app.js** — 初始化、控制面板事件、切换狠话/真心模式

## 3. 核心功能详细设计

### 3.1 Canvas 台阶绘制（核心视觉）
- **一级台阶**概念：绘制一条从左下延伸至右上的斜线阶梯，每层台阶承载一句语录
- **透视效果**：近处台阶较宽，远处台阶收窄，营造纵深感
- **文字呈现**：每级台阶顶面放置一句文字，文字有渐显动画，随台阶角度倾斜
- **动态移动**：台阶缓慢向上"滚动"，新台阶从底部出现，旧台阶从顶部消失

### 3.2 控制面板
- 模式切换：【狠话】/【真心】 两个按钮（高亮当前模式）
- 音乐开关 + 音量滑块（0-100）
- 特效强度滑块（低/中/高三档，或0-100连续值）
- 滚动速度滑块
- "随机一条"按钮：随机展示单条语录并放大突出

### 3.3 动态特效
- **粒子系统**：Canvas粒子从屏幕底部上升，模拟萤火虫/光点
- **发光效果**：文字阴影 `shadowBlur` 营造霓虹感
- **渐入渐出**：文字出现/消失的透明度动画
- **波纹**：切换语录时台阶边缘产生涟漪

### 3.4 背景音乐
- 使用 `<audio loop>`，提供默认播放按钮（浏览器策略要求用户交互后才能播放）
- 用户可上传本地音乐文件（`<input type="file" accept="audio/*">`）

### 3.5 现代UI设计要点
- 主色调：深紫 `#1a0b2e` → 深蓝 `#16213e` 渐变背景
- 点缀色：霓虹粉 `#ff2e9c`、电光蓝 `#00e5ff`、暖金 `#ffd86b`
- 控制面板：`backdrop-filter: blur(20px)` + 半透明白色边框
- 字体：`font-family: 'Segoe UI', 'PingFang SC', system-ui, sans-serif`
- 响应式：移动端控制面板折叠在底部，PC端悬浮右侧

## 5. 风险与注意事项

### 5.1 浏览器音频策略
- **风险**：现代浏览器（Chrome/Safari/Edge）禁止未触发用户交互时自动播放音频
- **对策**：页面首屏显示"点击开始"遮罩，用户点击后启动音乐和动画

### 5.2 Canvas性能
- **风险**：粒子过多或`requestAnimationFrame`内重计算导致低帧
- **对策**：粒子数设上限（如 ≤120），台阶几何数据缓存，离屏Canvas预渲染

### 5.3 资源缺失
- **风险**：背景音乐无默认文件
- **对策**：内置 Web Audio API 合成简单的环境音（低频和弦 + 轻柔噪声）作为默认BGM，无需外部文件；同时允许用户上传自己的音乐

### 5.4 移动端兼容性
- **风险**：`backdrop-filter` 在部分旧浏览器不支持
- **对策**：提供 fallback — 不支持时使用纯色半透明背景替代

✅ 控制面板可切换【狠话】/【真心】模式，切换后内容和色调变化
✅ 背景音乐可开关、可调节音量
✅ 动态粒子/发光特效可通过控制面板调节强度
✅ 整体视觉具有现代感（渐变、毛玻璃、霓虹点缀）
✅ 控制台无报错，60fps流畅运行
